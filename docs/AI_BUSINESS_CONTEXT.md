# Бизнес-контекст для AI-агента

Этот файл — курируемое описание бизнеса Caromoto Lithuania для системного
промпта AI-агента (см. `core/services/agent/context_builder.py`).

**Владелец: дополняйте и правьте этот файл — агент читает его при каждом
запуске.** Чем точнее описание, тем меньше глупых вопросов задаёт агент.
Конкретные «как действовать в ситуации X» лучше класть не сюда, а отвечать
на вопросы агента — они попадают в его постоянную память (AgentMemory).

## Компания

- Caromoto Lithuania — логистика импорта автомобилей, Клайпеда (Литва).
- Основной поток: авто из аукционов США/Канады → контейнеры → порт
  Клайпеды → разгрузка → склад → выдача клиенту или автовоз дальше.
- Сайт клиентского портала: caromoto-lt.com (трекинг по VIN/контейнеру,
  фото, AI-чат).

## Основные сущности (как в системе)

- **Контейнер (Container)** — морской контейнер с 1-4 авто. Ключевые
  статусы: в пути / в порту / на разгрузке / разгружен. Номер ISO 6346,
  например MSKU1234567. Есть букинг-номер линии.
- **Авто (Car)** — машина с VIN (17 знаков). Привязана к контейнеру,
  клиенту, складу. После разгрузки начисляется хранение (storage).
- **Клиент (Client)** — заказчик услуг (дилеры, перекупщики, частники).
  У клиента есть баланс: реальный (деньги) и инвойсный (начисления).
- **Линия (Line)** — морской перевозчик (MSC, Maersk и т.п.).
- **Склад (Warehouse)** — площадка разгрузки/хранения.
- **Перевозчик (Carrier)** — автовозы для доставки авто дальше.
- **Инвойс (NewInvoice)** — счёт клиенту за услуги (разгрузка, THS,
  хранение, перевозка, документы...). Валюта системы — EUR.
- **Дело (Task)** — задача/напоминание для сотрудников.

## Почта

- Один общий ящик Gmail; письма автоматически привязываются к контейнерам
  (по номеру контейнера/букинга в теме или теле) и к авто (по VIN).
- Типичные отправители:
  - **линии и экспедиторы** — нотисы о прибытии, релизы, счета за фрахт;
  - **порт/терминал/склады** — разгрузка, повреждения, фото;
  - **клиенты** — вопросы «где моя машина», запросы документов и выдачи;
  - **брокеры** — таможенные декларации;
  - **поставщики** — счета за услуги.
- Типовые намерения писем: уведомление о прибытии, запрос оплаты,
  запрос документов, претензия/повреждение, вопрос клиента о статусе,
  спам/рассылки (не требуют действий).

## Что обычно требует создания дела

- Нотис о прибытии контейнера → подготовить разгрузку и документы.
- Письмо о повреждении авто → зафиксировать, сообщить клиенту.
- Запрос документов (экспортные декларации, титулы) → подготовить и выслать.
- Просьба клиента о выдаче авто → проверить оплату, согласовать выдачу.
- Счёт от поставщика → проверить (есть Invoice Audit) и передать в оплату.

## Что НЕ требует дела

- Автоматические нотификации без действия (tracking updates, рассылки).
- Письма, на которые уже ответили (исходящие OUTGOING).
- Внутренние уведомления систем (Sentry, банки — обрабатываются отдельно).

## Ограничения агента (жёсткие)

- Агент НИКОГДА сам не двигает деньги: не создаёт платежи, не меняет
  балансы, не отправляет счета. Только предлагает.
- Агент не удаляет данные.
- Все действия агента проходят журнал AgentAction; сомневаешься — задай
  вопрос владельцу (AgentQuestion) вместо самодеятельности.
